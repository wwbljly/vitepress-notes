webpack操作 #

操作
1.初始化项目  npm init
2.安装webpack webpack-cli  webpack-dev-server,注:-D代表在当前项目下安装
	cnpm i webpack -D   cnpm i webpack-cli -D   cnpm i webpack-dev-server -D 

package.json配置:
	"webpack-dev": "webpack --mode development",
    "webpack-build": "webpack --mode production",
    "test": "echo \"Error: no test specified\" && exit 1",
    "watch": "webpack --mode development --watch",
    "serve": "webpack-dev-server --mode development",
    "dev": "webpack-dev-server --config ./build/webpack.dev.js",
    "build": "webpack --config ./build/webpack.prod.js"

webpack.config.js配置
const path=require("path");
const webpack=require("webpack");
const HtmlWebpackPlugin = require('html-webpack-plugin');
const {CleanWebpackPlugin}=require('clean-webpack-plugin')

module.exports={
    mode:"development",
    devtool:"cheap-module-eval-souce-map",
    devServer:{
        host:process.env.host,
        port:process.env.port,
        hot:true
    },
    entry:{
        index:path.resolve(__dirname,'./src/index.js'),
        other:path.resolve(__dirname,'./src/other.js')
    },
    output:{
        path:path.resolve(__dirname,'./dist'),
        filename:'[name].bundle.js'
    },
    plugins:[
        new CleanWebpackPlugin(),
        //模块热更新
        new webpack.HotModuleReplacementPlugin(),
        //提示变更的文件名
        new webpack.NamedChunksPlugin(),
      	//根据现有模板生成html
        new HtmlWebpackPlugin({
            filename:'index.html',
            template:'index.html',
            inject:true,
        }),
        new HtmlWebpackPlugin({
            filename:'index2.html',
            template:'index.html',
            inject:true,
            //在index2.html只引入other.js
            chunks:['other'],
        }),
    ],

    module:{
        rules:[
            {
                test:/\.css$/,
                use:['style-loader','css-loader']
            },
            {
                test:/\.scss$/,
                use:['style-loader','css-loader','sass-loader']
            },
            {
                test: /\.(png|jpg|jpe?g|gif|svg)(\?.*)?$/,
                loader: 'url-loader',
                options: {
                  limit: 10000,
                  name: 'img/[name].[hash:7].[ext]'
                }
            },
        ]
    }
}

webpack 构建流程分析 #

  1. 初始化:从配置文件和shell输入中读取参数,并合并得出最终参数
  2. 开始编译:根据参数初始化Compiler对象,加载并监听所有的插件,执行Compiler对象的run方法,开始执行编译
  3. 确定入口:根据配置中的entry,找出所有的入口文件
  4. 编译模块:从入口文件开始,根据文件类型和Loader配置,递归找出所有模块文件,对模块文件进行编译并得到模块之间的依赖关系
  5. 输出资源:根据入口文件和模块之间的依赖关系,生成一个个包含多个模块的chunk,再把每个chunk转换成一个单独的文件添加到输出列表
  6. 最后将文件输出到文件系统

Loader 和 Plugin 的区别 #

  • Loader主要是用来识别和分析非js文件,因为webpack自身只支持识别js文件,所以需要Loader来帮助识别其他类型的文件

  • Plugin是插件的意思,是用来扩展webpack的功能的,让webpack具有更多的灵活性

webpack热更新原理 #

热更新

webpack热更新其实是自身开启了一个express应用,添加了对Webpack编译的监听,并在服务器和浏览器之前通过webscoket进行连接,当文件变更触发webpack编译时,服务器通过webscokete向浏览器发送消息,浏览器此时会向webpack dev serve发送请求获取变更的文件信息以及文件的hash值,然后将新变更模块直接替换旧模块文件,从而不需要刷新页面而完成更新

webpack treeShaking机制的原理 #

treeShaking也叫摇树优化,主要作用就是通过移除多余代码来优化打包体积的

标记功能需要配置 optimization.usedExports = true 开启

  • make阶段:收集模块导出变量并记录到模块依赖关系图ModuleGraph中

  • seal阶段:遍历模块依赖关系图标记模块导出变量有没有被使用

  • 生成产物时:删除没有被使用的模块变量的导出语句

Babel 原理 #

  • 解析:将代码转换成AST语法树

  • 转换:访问AST语法树的节点进行代码转换并生成新的AST

  • 生成:根据新的AST语法树生成代码

webpack 有哪些常见的 loader #

cache-loader：可以在一些性能开销较大的 Loader 之前添加，目的是将结果缓存到磁盘里

file-loader：把文件输出到一个文件夹中，在代码中通过相对 URL 去引用输出的文件

url-loader：与file-loader 类似，区别是用户可以设置一个阈值，大于阈值会交给 file-loader，小于阈值时返回文件 base64 形式编码

image-loader：加载并且压缩图片文件。

babel-loader：把 ES6 转换成 ES5。

ts-loader：将 typescript 转换成 JavaScript

sass-loader：将 scss/sass 代码转换成 css。

css-loader：加载 css，支持模块化、压缩、文件导入等特性。

less-loader：将 less 代码转换成 css。

style-loader：生成 style 标签，将 js 中的样式资源插入，并添加到 header 中生效。

vue-loader：加载 vue.js 单文件组件。

source-map-loader：加载额外的 Source Map ⽂件，以⽅便断点调试

webpack如何实现持久化缓存 #

服务端设置http缓存头（cache-control）

延迟加载：使用import()方式，可以动态加载的文件分到独立的chunk,以得到自己的chunkhash

保持hash值的稳定：编译过程和文件内的更改尽量不影响其他文件hash的计算，

压缩代码：删除多余的代码、注释、简化代码的写法等等⽅式

利⽤CDN加速: 在构建过程中，将引⽤的静态资源路径修改为CDN上对应的路径

Tree Shaking: 将代码中永远不会⾛到的⽚段删除掉

提取公共第三⽅库: SplitChunksPlugin插件来进⾏公共模块抽取,利⽤浏览器缓存可以⻓期缓存这些⽆需频繁变动的公共代码

source map 是什么？生产环境怎么用？ #

source map 是将编译、打包、压缩后的代码映射会源代码的过程。打包压缩后的代码不具备良好的可读性，想要调试源码就需要 source map。

source-map：通过 nginx 设置将.map文件指对白名单开发。

文件指纹是什么 #

文件指纹是指打包后输出文件的名的后缀。

  • hash：和整个项目的构建相关，只要项目文件有修改，整个项目构建的 hash 就会变化。

  • chunkhash：和webpack打包的 chunk 有关，不同的 chunk、不同的 entry 会生成不同的 chunkhash。

  • contenthash：根据文件内容来定义 hash，文件内容不发生变化，则contenthash就不会变化
